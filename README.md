# ピコラボ 🎮

**曲をファミコンサウンドに変換して遊ぶラボ。** 手持ちの音源をブラウザに放り込むと、メロディ・ハモリ・ベース・ドラムの4トラック（ファミコン実機 2A03 と同じ編成）に分解して、ピコピコ鳴らし直します。

**▶ 遊ぶ: https://shimi-works.github.io/pico-lab/**

## できること

### ピコピコ再合成（メイン機能）
- mp3 / wav / m4a / ogg をドラッグ＆ドロップ → 解析 → ファミコン4トラックで再生
  - 🟢 メロディ（矩形波50%） / 🔵 ハモリ（矩形波25%） / 🟠 ベース（三角波） / ⬜ ドラム（15bit LFSRノイズ）
  - トラックごとにON/OFF、色分けピアノロール表示
- **リズム補正**: 曲のテンポを自動推定し、全ノートを16分グリッドに吸着＋レガート化
- 音源タイプ: 「曲・伴奏入り」（スペクトル解析）／「単音の録音」（YIN。鼻歌・口笛向け）
- WAV書き出し（モノラル・サンプルレート選択）

### 8bitエフェクト（サブ機能）
- ビットクラッシャー（ビット深度1〜8bit・サンプルレート低下・原音ミックス）＋ローパス
- プリセット5種（ファミコン風・ゲームボーイ風・壊れかけラジオ 等）＋ユーザープリセット保存

## プライバシーと著作権

- **すべてブラウザ内で処理。アップロードは一切ありません**（オフラインでも動きます）
- 入力はローカルファイルのみ。URLからの取得機能はあえて実装していません
- デモ音源は自作フレーズをその場で合成（音源ファイルの同梱なし）
- 既存曲を変換した音源の公開には権利者の許諾が必要です。自作曲でお楽しみください

## 技術構成

単一HTMLファイル（HTML/CSS/JSすべて内包・外部依存ゼロ）。

| 部品 | 実装 |
|---|---|
| 多声解析 | 自前FFT → HPSS（調波/打楽器音分離・メディアンフィルタ方式）→ 倍音サリエンス → Viterbi動的計画法によるピッチ軌跡追跡 |
| ハモリ抽出 | メロディの全倍音位置マスク＋「基音の実在×サリエンス」積で倍音ゴーストを排除した第2旋律線 |
| ドラム | 打撃成分のスペクトルフラックス → オンセット検出（帯域拡散・打撃比ゲート）→ キック/スネア/ハット分類 |
| 単音採譜 | YIN（CMND＋放物線補間）→ 連続ピッチセグメンテーション → 円周統計チューニング補正 |
| テンポ推定 | オンセットフラックスの自己相関（同点は細かいビート優先） |
| リアルタイム試聴 | AudioWorklet（Blob URL → file://では data: URL → 最終手段 ScriptProcessor の3段フォールバック） |
| 書き出し | 純関数DSP → OfflineAudioContext（フィルタ＋リサンプル）→ 16bit PCM WAV自前エンコード |

## 開発

```
node tests/test.mjs    # 純関数51件のユニットテスト（DSP・YIN・FFT・Viterbi・採譜・シンセ）
node tests/smoke.mjs   # ヘッドレスEdgeで file:// と http:// の両方をスモークテスト
```

DSP・採譜・シンセのロジックは `index.html` 内の `/* @pure-begin */ 〜 /* @pure-end */` に純関数として隔離してあり、テストがこの区間を抽出してNodeで実行します。
